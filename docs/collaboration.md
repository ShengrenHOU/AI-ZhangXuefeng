# Collaboration

## Repo vs External Docs

- repo docs store executable truth, contracts, setup, and governance
- discussion notes, meetings, and evolving strategy may live outside the repo
- published knowledge versions must remain in the repo for traceability

## Change Discipline

- keep filenames in English
- prefer small rule and schema changes over prompt-only tweaks
- update `docs/api-contract.md` and `docs/data-contract.md` when interfaces change
